## Phase 0: Research (Complete)

**Output**: [research.md](research.md) — AVM description style extraction and consolidation.

### Key Findings

1. **Canonical AVM bullet-list format**: `- \`field\` - (Required|Optional) Description.` — used consistently across newest modules
2. **"Deliberately arbitrary" map key statement**: Exact wording preserved from AVM: "maybe" (one word)
3. **Standard interface blocks**: lock (2-field), role_assignments (9-field), diagnostic_settings (10-field), private_endpoints (with ip_configurations sub-block)
4. **Copy-paste bugs to fix**: `<RESOURCE>` placeholder, "Key Vault" text in non-KV modules, "container app environment" in managed_identity
5. **managed_identity role_assignments is structurally different**: Uses `scope` instead of `principal_id`
6. **Storage account descriptions split across 6 AVM files**: Must merge from all variable files
7. **Inconsistent `(Required)`/`(Optional)` markers**: Normalize to always include on every leaf

## Phase 2: FR-026 — managed_identity_key Universalisation (Complete)

**Prerequisites**: Phase 1 complete, all 17 descriptions rewritten and validated.

### Summary

Extended `managed_identity_key` support from only `key_vaults` and standalone `role_assignments` to ALL resources with `role_assignments` blocks. Changes span `variables.tf` (types, descriptions, validations), `main.tf` (resolution logic), and example `variables.tf` (type mirroring).

### Key Decisions

1. **`resource_groups` excluded** — circular dependency: `module.resource_group` → `managed_identity_principal_ids` → `module.managed_identity` → `resource_group_names` → `module.resource_group`. Documented in pattern note.
2. **Hybrid validation approach** — XOR validation blocks on 8 top-level variables; description-only mutual exclusivity documentation on sub-resource blocks (queues, tables, shares, containers, private endpoints). Matches existing `key_vaults` pattern.
3. **`flowlog_configuration` null guard** — nullable object type requires `var.flowlog_configuration == null ? true : ...` in validation condition.

### Files Modified

| File | Change |
|------|--------|
| `variables.tf` | 19 type defs: `principal_id = optional(string)` + `managed_identity_key = optional(string)`. 8 XOR validation blocks added. 15 description blocks updated with mutual exclusivity docs. `resource_groups` reverted (circular dep). |
| `main.tf` | 9 top-level module role_assignments, 3 PE blocks, 7 sub-resource blocks: ternary resolution `ra.managed_identity_key != null ? local.managed_identity_principal_ids[ra.managed_identity_key] : ra.principal_id`. `resource_groups` left as pass-through. |
| `examples/*/variables.tf` | Type definitions mirrored from root (4 dirs × 13–19 blocks each). `resource_groups` kept as `principal_id = string`. |
| `README.md` (5 files) | Regenerated via `terraform-docs .` using `.terraform-docs.yml` configs. |
